## 🎯 核心定位

> 同学的可视化版本只展示"梯度下降"这一个点。
> 本 App 用游戏化的方式覆盖**整个第二讲 58 页 PPT**——包括同学版本完全没有的：
> 验证集 / L1+L2 / Dropout / Early Stopping / 数据增强 / 分类指标 / 网格搜索 / 高考一次性。

两个版本并列：

| 版本 | 入口 | 用途 |
|---|---|---|
| 🎣 **钓鱼游戏版** | `index.html` | 30 岁打工人自学，沉浸式过完全套 ML 概念 |
| 📐 **教学讲解版** | `classic.html` | 老师上课用的同学原作 + URL hash 同步增强 |

## 🏗 三阶段游戏流程

按 PPT 第 12 页阶段小结表的 **"人 vs 机器"** 分工设计：

```
① 渔具店 [👤 人决定]  →  ② 钓鱼场 [🤖 机器训练]  →  ③ 战绩评定 [👤 人评判]
```

### ① 渔具店（配置超参数）

按 PPT 的"超参数三大类"严格分组的 5 个 shop-card：

| Card | 中文 | PPT 出处 | 可调装备 |
|---|---|---|---|
| 🗺️ A | 数据准备 | 超参数 (1) | 鱼塘地图、鱼群规模、噪声、**数据三分比例**、数据增强 |
| 🎣 B | 模型设定 | 超参数 (2) | 鱼竿节数（多项式次数 1-15） |
| ⚙️ C | 模型训练 | 超参数 (3) | 学习率 η、max epoch、早停 |
| 🛡️ D | 防过拟合 | PPT 第 17-19 页 | L1 / L2 / Dropout |
| 🎯 E | 评判标准 | 性能指标 | 主 KPI（MSE / Accuracy / Precision / Recall / F1）+ 钓中阈值 |

**辅助功能**：
- **🤖 自动网格搜索 Banner**：跑 12 组超参组合，按模拟考损失排名，点"采用"自动填入
- **4 个一键预设**：🌱 新手村 / 🌀 弯曲挑战 / 🪤 过拟合诊所 / 🛡️ 防御工事

### ② 钓鱼场（梯度下降训练）

**Canvas 三色鱼可视化**：

| 颜色 | 标记 | 角色 | 训练时 |
|---|---|---|---|
| 🟧 训练鱼 | 彩色 | 练习池 (~70%) | 参与梯度下降 |
| 🟪 验证鱼 | 紫 + `V` | 模拟考池 (~15%) | 早停判断 + 实时显示模拟考损失 |
| 🔴 测试鱼 | 红 + `?` 锁 | 高考池 (~15%) | **完全锁定**，灰雾显示 |

**实时面板**：
- 5 个指标卡：训练损失 / 模拟考损失 / 🔒高考损失 / epoch / F1
- 双损失曲线：橙色训练损失 + 紫虚线模拟考损失（**PPT 第 13-16 页学习/验证曲线**）
- 现场可调：学习率、早停开关
- 喵老板智能提示：检测到过拟合 / 学习率发散 / 早停触发会自动播报

### ③ 战绩评定（按 PPT 性能指标评估）

**段位评语**：CTO 喵 → P8 高级喵 → 高级钓鱼师 → 初级钓鱼师 → 正式员工 → 实习生 → 摸鱼候选人

**Tab 切换**：

#### 📝 模拟考成绩（可多看）
- 4 大指标卡：Accuracy / Precision / Recall / F1（PPT 第 47-53 页）
- 混淆矩阵 TP / FP / FN / TN（PPT 第 49-50 页）

#### 🎓 高考成绩（一次性）
- 默认显示**锁定页 + 📢 大按钮**
- 点公布按钮 → 立即拍照保存权重快照 → 用快照算高考指标 → 按钮变灰永久禁用
- 公布后展示：4 大指标 + 混淆矩阵 + **泛化能力诊断**

**泛化能力诊断**（v4 独家）：自动对比 `测试集损失` vs `验证集损失`：
- 差距 < 0.5 → ✅ 泛化良好
- 测试 > 验证 0.5+ → ⚠️ "你在调超参时过拟合到了模拟考池"
- 测试 < 验证 → 😯 运气好或模拟考太难

## 🤖 自动网格搜索（v4 新增）

PPT 字幕里讲超参选择本质是"试错"——跟梯度下降同构，只是用"性能指标"代替"损失函数"。
本 App 把这个抽象观点**做成了真实可点的按钮**：

**搜索网格**（12 组预设组合）：

| # | degree | η | L2 | dropout | early stop |
|---|---|---|---|---|---|
| 1 | 1 | 0.02 | 0 | 0 | ✗ |
| 2 | 2 | 0.02 | 0 | 0 | ✗ |
| ... | ... | ... | ... | ... | ... |
| 12 | 15 | 0.02 | 0.2 | 0.5 | ✓ |

每组在模拟考池上跑 300 epoch，按验证损失升序排名。每行带 `[采用]` 按钮，自动填入配置。

**对应 ML 术语**：Grid Search · Validation-based Hyperparameter Tuning

## ⚙️ 技术栈

| 维度 | 选型 |
|---|---|
| 架构 | 单文件 HTML，**零依赖**，纯 Canvas 2D + DOM |
| 模型 | 多项式回归（次数 1-15 动态权重数组） |
| 训练核心 | 梯度下降 + L1/L2 正则项 + Dropout + Early Stopping |
| 指标计算 | MSE + 阈值化二分类 → Accuracy / Precision / Recall / F1 |
| 自动搜索 | 12 组预设网格 · 异步分批 setTimeout · 排名展示 |
| 数据生成 | 三池随机分配（70/15/15 默认）+ 数据增强（镜像 x） |
| 跨页同步 | URL hash 传 `?a=2.5&b=1.4` 实现 v4 ↔ classic 互通 |
| 部署 | 腾讯云 1Panel + OpenResty + Let's Encrypt HTTPS |
| 大小 | **93 KB 单文件** |

## 💡 推荐试玩流程（30 秒理解"测试集为什么只能碰一次"）

1. 打开 https://ai.hutiefang.com/
2. 渔具店点 `🪤 过拟合诊所` 预设
3. 点 `🤖 开始自动搜索 12 组`，等约 10 秒
4. 看排名列表，点 `#1` 的 `[采用]`
5. 点 `🎣 出海钓鱼`，自动训练完成
6. 点 `📊 看战绩` → 进战绩页
7. 切到 `🎓 高考` Tab → 看到锁定页
8. 点 `📢 公布高考成绩`
9. **泛化诊断会告诉你"你过拟合到模拟考池了"**——这正是 PPT 第 13 页字幕讲的"测试集只能碰一次"的根本原因
